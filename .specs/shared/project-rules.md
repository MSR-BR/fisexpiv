# Project Rules

## Regras de escopo

- Implementar somente a mudança aprovada.
- Não antecipar features futuras dentro do MVP.
- Tratar simuladores como incrementos separados quando exigirem esforço relevante.

## Regras de conteúdo

- Nenhuma prática será considerada pronta sem fonte identificável.
- Textos pedagógicos devem ser revisáveis e rastreáveis.
- Material antigo pode servir como base, mas não deve ser copiado cegamente.

## Regras de estrutura

- Toda prática deve seguir o mesmo esqueleto pedagógico base.
- A home deve funcionar como ponto central de orientação.
- O menu geral deve ser simples, estável e fácil de expandir.

## Regras de rastreabilidade

- Cada mudança deve listar arquivos previstos.
- Toda decisão importante deve estar registrada em `.specs/`.
- Assunções devem ser escritas claramente quando faltarem dados.

## Regras de aprovação

- A `Change 001` só começa após aprovação explícita do usuário.
- Mudanças seguintes dependem de revisão do resultado anterior e de eventuais novos insumos.
