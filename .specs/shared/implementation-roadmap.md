# Implementation Roadmap

## Fase 0 - Especificação

- consolidar arquitetura;
- definir padrões;
- organizar changes;
- explicitar assunções e dependências.

## Fase 1 - Fundação técnica

- iniciar projeto Astro;
- criar estrutura de pastas;
- configurar TypeScript, lint e testes;
- preparar schema de conteúdo.

## Fase 2 - MVP estrutural

- construir home;
- implementar navegação global;
- criar template de prática;
- habilitar conteúdo MDX por prática.
- publicar o menu completo com todas as nove práticas.

## Fase 3 - Primeira prática completa

- modelar as práticas `Revisão` e `Carga-massa`;
- validar estrutura pedagógica;
- revisar fluxo de aquisição e análise de dados;
- ajustar componentes reutilizáveis.

## Fase 4 - Simuladores e blocos interativos

- definir padrão de integração;
- conectar simulador interno ou externo validado;
- suportar tabelas, parâmetros e visualização de resultados.

## Fase 5 - Validação e publicação

- rodar testes automatizados;
- revisar conteúdo e links;
- preparar build estático;
- publicar ambiente inicial.

## Dependências críticas

- material fonte validado;
- definição dos simuladores prioritários.
